# Adversarial stress-test of §16 (the one neuro-endocrine-oscillatory loop)

## (1) One loop, or three bolt-ons? — the couplings that are asserted but not mechanistically real

The honest verdict: the **control** couplings are genuinely one loop, but **three of the named arrows in the 16.1 diagram do not correspond to a real signal path in the current code**, and one whole layer (dynamics/ignition) is separable enough to be a bolt-on.

What *is* real and shared (so this is not three independent subsystems): every endocrine output lands on a *small set of already-existing scalars* — `gate` (`life.py:294`), `self.debt` (`:747/689`), the reward handed to `bg.train_step` (`:827`), and `self.attention`. `C→gate` via `g(C)`, `C→debt`, and `r_home→reward` are all trivially real (each target is a scalar multiply/add that already exists). That convergence onto shared state is what makes it a loop rather than three stacks. Good.

But four seams are weaker than the prose claims:

- **`salient hippo key → recall gist → generate() dream` — the seed carries almost no information.** The hippocampal "gist" is *not* a latent trace; `_fingerprint` (`life.py:795–805`) is a normalized **256-bin byte-frequency histogram**, and `hippo.store` keys/values are that histogram (`spiking_modules.py:196–200`). `recall` returns `w @ self.vals` (`:223`) — a weighted average of byte histograms. So the "reconstructed gist" you condition the dream on is a **bag-of-bytes letter-frequency vector with zero sequential/semantic content**. Conditioning `generate()` on it (as a "logit bias") can only skew *which bytes* appear, not *what is said*. The CLS claim "hippocampus reinstates, cortex integrates" is asserted, but the reinstatement channel physically cannot transmit an episode. This is the single least-real coupling in the design.

- **`generate()` has no conditioning port.** `generate(prompt, …)` (`spiking_brain.py:585`) conditions only on a *byte-id prompt*, building state via `self._run(x)`. There is no entry point to inject a 256-vector as initial state (the real state is per-cell `(v,s)` tuples of width = hidden dim, thousands wide) or as a logit bias. The "project the gist as an initial-state/logit-bias seed" is an unbuilt hook over a signal (previous bullet) that has nothing to project. Both must be *invented*, and even then the information content is a letter histogram.

- **`up ∧ spindle → consolidation-commit` is a scheduler, not the eligibility-coincidence gate it's dressed as.** e-prop here has **no persistent cross-tick eligibility trace to gate**: `eps_rec/eps_in` live *inside a single `learn_text` call* over its T timesteps and the update is applied at the end of that call (`_eprop_step`, `spiking_brain.py:396–397, 480–492`). There is no standing trace between sleep ticks that a SO up-state could "commit." So "commit only in up∧spindle windows" reduces to "call `learn_text` on some ticks, not others" — fine as a cadence, but it is **not** the biological ripple→spindle→SO timing of when plasticity commits that the section sells (Staresina-style). Asserted mechanism ≠ implemented mechanism.

- **Half the "operating point" is inert at the default config.** `diff_neuromod = False` by default (`spiking_brain.py:141`). With it off, NE/5-HT/DA tone do **nothing** to the cortex (`diffnm` guards at `:337–341, 383, 395`); the *only* endocrine handle on cortical plasticity is the ACh `gate` and `attention`. So "ΣD→NE exploration-temperature," "mood→5-HT→patience," and the DA-magnitude arm are latent — the rich "controller over four tones" collapses to **two scalars** (gate, attention) unless you first enable `diff_neuromod`, which itself changes cortex behavior and would need re-validation before you build on it.

And the **dynamics/ignition layer is the most bolt-on-ish piece**: it shares only β (derived from NE+attention) and, unlike the endocrine↔memory arms, it does **not feed back into learning** except by skipping modules. You could delete the entire ignition sublayer and the endocrine→sleep→consolidation→mood loop would be unchanged. If any part is a detachable add-on, it is selective-activation — which is also (see FM3) the lowest-value part.

## (2) Top 3 failure modes when built

**FM1 — Generative-replay regresses *below* the current buffer brain (model-autophagy).** This is the dominant risk and it is structural, not a tuning issue. The buffer, however "biologically wrong," is a *correct* anti-forgetting signal today. You propose to demote it and train on `generate()` output seeded by a letter histogram (Q1). At this stage the cortex is a small, continually-trained byte RNN; if its samples are repetitive/off-manifold (exactly what `generate_diag`'s `entropy_bits`/`perplexity_gen` at `spiking_brain.py:630` exist to detect), then each night you (a) hard-learn garbage, (b) then apply the SHY downscale `p.mul_(1−2e-5)` (`life.py:934`) on top, eroding the very topic-A weights you meant to protect. The dream→learn→dream loop is a positive feedback toward a low-entropy attractor. The design's §16.3 safeguards are the right instinct, but the *ordering* is wrong: **don't demote the buffer before the generator has earned its share** (fix in Q3/Q4).

**FM2 — Cortisol × attention form a second negative-feedback loop on the same plasticity scale → stress latch or limit-cycle.** The cortex *already* self-regulates plasticity: `surprise=(L−loss_ema)/…` → `attention↓` → `scale=gate·lr·attention·(0.5+da)` (`spiking_brain.py:474–479`). You now add a *second* controller on the same `scale`: `surprise→C↑` → `g(C)↓` → `gate↓`. Both are negative feedback, driven by the *same* surprise signal, with *different lags* (attention is a 0.9/0.1 EMA per call; C has τ≈one wake-bout). Two lagged controllers fighting over one actuator is the textbook recipe for a **limit cycle in the learning rate**, or a **latch into the low-plasticity/high-C corner**: attention↓ and gate↓ keep loss high, keeping surprise high, keeping C climbing. The stabilizers you cite are *slower* than the destabilizer — mood damps C only through an λ≈0.02 EMA of δ_DA, and sleep relief only arrives when `debt≥6` or `max_awake=300s` trips (`life.py:113, 890–895`). Loop gain favors the spiral. Secondary effect: `debt += base·(1+w·C)` with C=O(1) roughly halves the wake bout under stress → **sleep fragmentation** (frequent short nights), bounded only by `min_awake=90s`. Fix: one arbitrator, not two writers — have C modulate the *setpoint/sensitivity* of the attention update (e.g. `attn_sensitivity` or the clamp), never multiply the same `gate` the attention loop already owns; cap `dC/dt`; guarantee relief-τ ≤ accumulation-τ.

**FM3 — Ignition gating costs more than it saves at 256k *and* can truncate e-prop credit.** The compute argument is backwards for this architecture. The 256k **cortex dominates cost and already runs `think()` every tick** (`life.py:990`); the four side-modules are tiny (`n_granule=2000`, `n_msn=64`, DG store) and **already event-gated** — cerebellum/bg/hippo only fire inside `_consume_perceptions` when a teach/browse arrives (`:783–785`). So β-gated ignition of the four cheap modules saves ≈nothing, while adding a per-tick softmax/phase bookkeeping overhead. Gating where the cost actually is — the cortex — means skipping `think()`/`learn` on low "surprise" (drive = `1−attention`), i.e. **you stop consolidating exactly when the brain is doing well**, and you do it using the *same* signal the attention loop already consumes (double-coupling → adds to FM2). Worse, the attention→frequency→`eff_β` arm sets the eligibility decay `eb` (`spiking_brain.py:394`): under focus you *shorten* the window, and if `eff_β` drops the effective trace length below the sequence length T (`seq` up to 96), you **truncate credit assignment for long-range byte dependencies** — a direct, silent harm to the one learning rule that works. Note also `attention∈[0.2,1.3]` (`:476`), so `f_m=f_α+A·(f_γ−f_α)` with A=1.3 overshoots past gamma — the map assumes A∈[0,1].

## (3) Is P0 the best first prototype? — a sharper P0

P0 (generative-replay-replaces-buffer) is the right *headline* and it is independently testable, but as scoped it is **neither the most self-contained nor the safest first step**, for three reasons: it depends on two unbuilt, low-information pieces (the gist and the conditioning port, Q1), its value is contingent on generation quality the from-scratch brain may not have yet (FM1), and "demote the buffer" makes the first prototype able to **regress the currently stable brain**. "Self-contained" in the design means "no endocrine/dynamics," but it is *tightly* coupled to hippocampus internals (needs per-key salience/recency metadata that `store` does not keep — it is a bare ring buffer, `:196–200`) and to a generation-quality assumption.

**Better P0 (a strict superset that cannot regress): interleaved generative replay with the buffer kept as a veridical anchor, gated by held-out bpb.** Concretely: keep `_sleep_tick` training on buffer samples, but *add* a generative-dream fraction f; measure every dream batch with `generate_diag` + `bits_per_byte(HELDOUT_PROBE)` and **discard batches that fail** (skip their SHY downscale too); a slow controller raises the anchor fraction toward 1 when discard-rate is high. This is exactly van de Ven's prescription in `runs/memory_architecture_research.md`, and its safety property is decisive: **if the generator is bad, f→0 and you are provably back to the current stable brain**; if it's good, you get the buffer-free benefit and can *then* demote the buffer in P0.5. Same headline, same forgetting-resistance test, zero regression floor.

If you want an even safer, *fully* self-contained first brick that tests the user's "memory lives in the net" thesis **without** any generator or hippocampus dependency: **importance-gated multi-timescale plasticity (Benna–Fusi / Synaptic Intelligence, Survey 4).** It is purely local to `_eprop_step` — one slow per-weight variable plus a leak in `_upd` (`spiking_brain.py:480–481`) and an online importance that scales `scale` per-weight — needs no dream loop, no seed, no sleep rewrite, cannot collapse, and is verified with the *identical* A→B–E forgetting protocol (expecting power-law, not exponential, decay). I would run this **in parallel as P0-companion**: it de-risks the whole memory thesis and, if generative replay stalls on generation quality, it independently delivers "generalized-in-the-net" retention. My recommendation: **P0 = interleaved-anchored generative replay (superset test), with importance-gated plasticity as the parallel safety net; demote the buffer only in P0.5 after the held-out gate has held for ≥5 nights.**

## (4) Precise P0 acceptance test — proves it works *and* doesn't degrade the stable brain

Run three arms with **identical seed, curriculum, and config**, differing only in the sleep training source: **(i) raw-buffer** (current), **(ii) anchored-generative** (P0), **(iii) no-replay** (SHY-only, no training source).

**First, capture the stable-brain baseline band** (arm i, current default) over K≥5 nights: `HELDOUT_PROBE` bpb trajectory (`life.py:44`, via `bits_per_byte`), `next_byte_acc(probe)`, `gen_entropy`, `spike_rate`, and `weight_stats` mean|w|/dead-fraction. These define the non-regression bands.

**Headline claim — forgetting-resistance.** Pin the feed (disable the Sonnet teacher, fix a topic-A corpus): train **only A** for N wake-bouts, snapshot `acc_A0 = next_byte_acc(A_probe)`. Then train B,C,D,E sequentially, N bouts each, with **no A data**. Measure `acc_A_final`; retention = `acc_A_final/acc_A0`.
- **Accept iff** retention(ii) ≥ retention(i) − ε *and* both ≫ retention(iii). (Generative must match or beat the buffer; both must beat none.)
- **Stronger "in-the-net" test:** with `memory` fully disabled as a *training input*, A-probe accuracy must still *rise* across nights under arm (ii) — proving the generalized memory lives in the weights, not the deque.

**Non-degradation guardrails — ALL must hold, or reject even if retention passes:**
- `HELDOUT_PROBE` bpb end-state ≤ baseline + small band, and **non-increasing across ≥5 nights** (the van de Ven anti-autophagy check; a rising cross-domain bpb is the collapse signature).
- `gen_entropy` stays inside `[floor, ceiling]` every night — →0 = repetition collapse, →8 = noise; reject on either.
- `spike_rate` stays in its healthy band (not →0 dead, not saturated); `weight_stats` mean|w| not runaway (SHY still nets ⟨Δw⟩≈0 — the `p.mul_(1−2e-5)` must be applied **only on accepted batches**).
- **Wall-clock/night** for arm (ii) within X% of arm (i) — `generate()` is autoregressive n-steps; verify the dream cost is bounded at 256k.
- **Safe-rollback / inertness:** with the P0 flag OFF, arm (ii) must reproduce arm (i)'s metrics within noise for M nights — proving the change is inert when disabled and the current brain is untouched by the code path.

**In-loop acceptance gate (this is what guarantees non-regression, not just measures it):** each dream batch → `generate_diag` entropy + `bits_per_byte(HELDOUT_PROBE)` pre/post; if entropy < floor **or** held-out bpb rises, **discard the batch and skip its SHY downscale**; if the nightly discard-rate exceeds a threshold, **auto-raise the anchor (buffer) fraction**. In the limit this recovers the current stable brain — so the worst case of P0 is "no worse than today," which is precisely the property a first prototype on a running, self-tuning brain must have.

Files that must change for P0 (minimal): `brain/spiking_modules.py` (`SpikingHippocampus.sample_key` + per-key recency/novelty metadata — `store` currently keeps none), `brain/spiking_brain.py` (a real conditioning entry to `generate` — and be honest that a 256-bin byte histogram is a weak seed; consider seeding from a short recalled *byte prompt* instead), `brain/life.py` (`_sleep_tick` anchored-dream loop + the held-out accept gate; `_net_diag` logging of dream-accept-rate and anchor-fraction). Do **not** touch the endocrine/dynamics surface for P0.
